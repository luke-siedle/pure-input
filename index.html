NOTE: Please run this documentation on your local server, or some of the examples featuring AJAX may not work correctly.

Basic Topics

Introduction

Pure is a plugin library built on jQuery and John Resig's JavaScript inheritance pattern, for enhancing native browser elements.

Installation

Simply add the pure-base-1.0.min.js file and pure-input-1.0.min.js file in the head of your document just after you include jQuery. For styling, begin by using the default css template.

					<html>
						<head>
							<title>My Document</title>
							<script type="text/javascript" src="js/jquery-1.7.2.min.js" ></script>
							<script type="text/javascript" src="js/pure-base-1.0.0.min.js" ></script>
							<script type="text/javascript" src="js/pure-input-1.0.0.min.js" ></script>
							<link rel="stylesheet" href="css/pure-default-1.0.0.min.css" />
						</head>
					</html>

Thereafter, construction of a Pure object should be nested within a jQuery ready handler.

					
					// Wait for the input element to be available in the DOM //

					$( function(){

						$('.myInput').Pure('Input', {
							hint : 'I am ready'
						});

					});

For the purposes of this documentation, any JavaScript code is assumed to be nested within a jQuery ready handler.

Implementation

Implementation uses only a single jQuery namespace, "Pure". There is the Pure object you want to call, in this case Input, the set of user-defined options to go with it, and a public interface to give you access to some useful methods.

					
					//	Simple jQuery binding //

					$('.myInput').Pure('Input', {
						hint : 'Focus on me'
					});

					/*
					*	Note : The above method of construction 
					*	supports chaining but in this case, 
					*	Pure hides the original input and uses 
					*	its own one, since it has to complete 
					*	more complex functions and be available 
					*	to move flexibly in the DOM.
					*/

					// Simple Object Oriented binding //

					var input = new Pure('Input', {
						el		: $('.myInput'),
						hint	: 'Focus on me'
					});

					/*
					*	Note : If you use the OOP methodology as above,
					*	you are automatically returned
					*	a public interface of methods.
					*/
					
					input.val('New value');
					input.focus();
					
					/*
					*	If you instantiate using jQuery, you can 
					*	access the methods by using the same syntax
					*	as you used to construct, but without passing
					*	any options.
					*/

					var methods = $('.myInput').Pure('Input');

					methods.val('Another new value');
					methods.focus();

					methods == input // true //
You can add plugins either during instantiation (in the options object), or at any time in the script. 
					
					//	At instantiation //

					var $el = $('.myInput').Pure('Input', {
						plugins : {
							Autocomplete : {
								src	: 'ajax.php'
							}
						}
					});

					// After instantiation //

					$('.myInput').Pure('Input:Autocomplete', {
						src : 'ajax.php'
					});
Over and above the public methods interface, you have access to certain listeners, that is, methods that are executed when certain events occur. 

					//	At instantiation //

					var $el = $('.myInput').Pure('Input', {
						on	: {
							// Listens for a change in value //
							changeVal : function( val ){
								alert('Value is different!');
							}
						}
					});

					// Overwrite the method after instantiation //
					
					$('.myInput').Pure('Input', {
						on	: {
							// The listener always has the context (this)
							// of the public interface
							changeVal : function( val ){
								if( val == 'Hulk' ){
									this.destroy();
								}
							}
						}
					});

Styling

Pure input inherits the class names you give to the input element. You may want to custom style the other plugins, like Autocomplete. Consult the pure-input-1.0.css stylesheet which contains comments you can follow to custom-style these plugins.

API Documentation

Global Methods

Each plugin contains the following methods in its public interface.

Method Arguments Returns
options [ options | string ] Context or Option(s)
Get or set the options object of the plugin.

options - overwrites the specified options with new ones.

string - returns a specific option.
attr [ attribute, value ] value
Context
Get the attribute of the input (if second argument is null).

attribute - the attribute to get or set.

value - the value to apply to the attribute.

Supported attributes are plugin specific. This method may be redundant in some cases.
get none plugin instance

Get the actual instance of the plugin, for deeper manipulation.
destroy none true

Destroy the instance, removing any events and elements that were created for its purpose. Restores the element it was applied to to its former state (if applicable).

Input

Options

Option Type Default
hint String
An initial "value" to show when input is awaiting use.
value String
Initial value of input.
height Integer 20
The height the actual input element will be, even while growing. Usually near to css font-size.
autogrow Boolean false
The input may grow dynamically as the value changes.
minWidth Integer 10
The smallest width the Input can be if its size is dynamic.
maxWidth Integer
The maximum width the Input can be if its size is dynamic.
maxHeight Integer
The maximum height the Input can be if its size is dynamic. Thereafter, it will become scrollable.
maxLength Integer
The maximum number of characters.
plugins Object
An object listing associated plugins and their respective options.
on Object
An object listing associated listeners and their behaviour. 
					// Naturally, you group these options
					// together in a single object...

					var options = {
						hint		: 'Type to start searching...',
						value		: 'This is an initial value',
						height		: 20,
						autogrow	: true,
						minWidth	: 10,
						maxHeight	: 100,
						maxWidth	: 200,
						maxLength 	: 2000,
						plugins		: {
							Autocomplete : { /* Options */ }
						},
						listeners	: {
							on		: changeVal( ){}
						}
					};

					// ...and apply them to your instance construction //

					$( 'input.superVillains' ).Pure( 'Input', options );

Listeners

Listener Arguments Returns
change [value] Context
Listens for a change in the input's value.

value - the current value of the input. Occurs on each keypress event.

Example:


								// Listen for a change in value and log it //

								$( 'input.superVillains' ).Pure( 'Input', {
									on : {
										change : function( val ){
											console.log( val );
										}
									}
								});

Methods

First get back the methods interface: 


					// Construct the Input //

					$( 'input.superHeroes' ).Pure( 'Input', {
						hint : 'Superheroes'
					});

					
					// Access the public methods interface //

					var input = $( 'input.superHeroes' ).Pure( 'Input' );

Then you can execute a method: 

					input[ methodName ]( handler );
Method Arguments Returns
focus [ handler ] Context
Focus on the input (if arguments are empty).

handler - run this code on focus.
blur [ handler ] Context
Blur the input (if arguments are empty).

handler - run this code on blur.
val [ value ] value
Context
Get the value of the input (if arguments are empty).

value - set the value.
setHint [ hint ]
Sets the value of the hint and renders it. Overwrites the hint option.

hint - the string to use as a hint.
attr [ attribute, value ] value
Context
Get the attribute of the input (if second argument is null).

attribute - the attribute to get or set.

value - the value to apply to the attribute.

Supported attributes:

disabled, readonly

Should you wish to manipulate the plugin on a deeper level, you can access the instance using .get() 


					// Get the Input instance //

					var inst	= input.get(),
						view	= inst.view;
					
					// Access the parent Input element (jQuery) //

					console.log( view.el );


					// Access the Input's elements //

					console.log( view.elements );

Examples

						$( 'input.__example1').Pure('Input', {
							hint		: 'Describe your personality in 300 words',
							height 		: 20,
							autogrow 	: true,
							maxHeight 	: 100,
							on			: {
								change : function( val ){

									if( val.length > 300 && ! alerted ){
										alert('Okay, okay!');
										alerted = true;
										this.attr('disabled', true );
										selfDestruct( this );
									}

									function selfDestruct( input ){
										input.val('This input will self destruct in 3 seconds.');
										setTimeout( function(){
											input.destroy();
										}, 3000 );
									}
								}
							}
						});

						var methods = $( '.__example1').Pure('Input');

						methods.focus ( function(){
							this.setHint('The input will disable and self-destruct\
											when you reach 300 characters.');
						});

Input : Autocomplete

Options

Option Type Default
src String, Function, Array

The source to use for results. The source can either be a string indicating a URL, a function which will create the results, or an array of local results.

Example:


							// As a String //

							$( 'input.superHeroes').Pure('Input:Autocomplete', {
								src : 'ajax.php?type=superheroes'
							});


							// As a function for result delivery //

							$( 'input.superHeroes').Pure('Input:Autocomplete', {
								src : function( searchVal, onComplete ){

									// Perform sync or async tasks
									// Then run onComplete

									var results = [];
									onComplete( results );
								}
							});

							var localResults = [];

							$( 'input.superHeroes').Pure('Input:Autocomplete', {
								src : localResults
							});

Dependencies

You should configure the mapSrc function as indicated below.
mapSrc Function

This function is required to parse each result to allow for templating. The function will be run through each iteration of the results, returning the necessary data to create the template. You must always return an object with an id and value.

Example:


								$( 'input.superHeroes').Pure('Input:Autocomplete', {
									mapSrc		: function( i ){
										return {
											id 		: i.userID,
											value	: i.fullName
											/* More fields... */
										};
									}
								});

								// mapSrc is now used in iterating
								// through a result set like this one
								var results = [{
									userID		: 125542354,
									fullName	: 'Darth Maul'
								}];
mapResults Object

This object is not necessary unless you are returning a multi-dimensional result set. It allows you to specify a categorized result set, and what to title each category.

Example:


								$( 'input.superHeroes').Pure('Input:Autocomplete', {
									mapResults : {
										family	: 'Family',
										friends : 'Friends'
									}
								});

								// This map now expects a result like this one //

								var result = {
									family : [{
										name	: 'Darth Vader'
									],
									friends : [{
										name	: 'Han Solo'
									}]
								}
minResults Integer 30
The minimum number of results required to skip the running of an asynchronous call (if applicable).
maxResults Integer 30
The maximum number of results to show at any given time.
searchTimeout Integer 500
The length of the delay in milliseconds the plugin will wait after the last key press before it begins searching asynchronously (if applicable).
templates Object Object (see below)

The templates object contains the strings necessary for output, and can be user-defined.

Example: (defaults)

								$( 'input.superHeroes').Pure('Input:Autocomplete', {
									templates 	: {

										// Different states of Autocomplete //
										state 	: {
											'start' 				: 'Type to start searching...',
											'startMention' 			: 'Select an item to mention...',
											'searching' 			: 'Searching...',
											'tokenizer_delete'		: 'Press Backspace or Delete to remove',
											'token_limit_reached'	: 'Selection limit reached.\
																		Delete items to continue.'
										},

										// The result item //
										result	: '{value}',

										// The result category (if applicable) //
										category : '{value}'
									}
								});
on Object
An object listing associated listeners and their behaviour.

Listeners

Listener Arguments
listItemSelect [ listItem ]
Listens for the selection of a list item.

listItem - an object containing the result data (parsed by options.mapSrc).

Example:


								// Listen for the selection of data and log it //

								$( 'input.superVillains' ).Pure( 'Input:Autocomplete', {
									on : {
										listItemSelect : function( listItem ){
											console.log( listItem );
										}
									}
								});
results [ type, resultSet ]
Listens for results found either locally or remotely.

type - a String representing the type of transaction ('sync', 'async').

resultSet - an object containing the data that was found during the transaction.

Example:


								// Listen for the discovery of results and log it //

								$( 'input.superVillains' ).Pure( 'Input:Autocomplete', {
									on : {
										results : function( type, resultSet ){
											console.log( type, resultSet );
										}
									}
								});
noResults [ type ]
Listens for empty results either locally or remotely.

type - a String representing the type of transaction ('sync', 'async').

Example:


								// Listen for failure to find results and log it //

								$( 'input.superVillains' ).Pure( 'Input:Autocomplete', {
									on : {
										noResults : function( type ){
											console.log( type );
										}
									}
								});

Examples

					
						// The initial Input binding //

						$( 'input.__example2').Pure('Input', {
							hint		: 'Search for "Batman"',
							height 		: 20,
							autogrow 	: true,
							maxHeight 	: 100
						});

						// A local source of results //

						var src = [
							{ id : 1, name	: 'Batman Returns (1992)'},
							{ id : 2, name	: 'Batman Forever (1995)'},
							{ id : 3, name	: 'Batman and Robin (1997)'},
							{ id : 4, name	: 'Batman Begins (2005)' }
						];

						// The default set of options for autocomplete //
						
						var opts = {
							src			: src,
							mapSrc		: function( i ){
								return {
									id : i.id,
									value : i.name
								};
							},
							on : {
								listItemSelect : function( listItem ){
									alert( 'You chose ' + listItem.name );
								}
							}
						};
						
						// Bind the options //

						$( 'input.__example2').Pure('Input:Autocomplete', opts );

						// Get the methods interface //

						var auto = $( 'input.__example2').Pure('Input:Autocomplete');

						
						// Buttons behaviour //

						$('button.__example2.disable').click( function(){
							auto.destroy();
						});

						$('button.__example2.enable').click( function(){
							$( 'input.__example2').Pure('Input:Autocomplete', opts );
							auto = $( 'input.__example2').Pure('Input:Autocomplete');
						});

Expanding on this example, we allow for the ability to change the source, by reassigning the options and using the history functions. 

						
						// A source handler that uses the Facebook Graph API and JSONP //

						var fbSrc = function( val, onComplete ){

							var callback = 'fbcallback' + new Date().getTime();
							$.getScript( 'https://graph.facebook.com/search?q='
											+ val + '&callback=' + callback + '&type=page' );

							window[ callback ] = function( response ){
								onComplete( response.data );
								window[ callback ] = undefined;
							}
							
						};
						
						$( 'input.__example3').Pure('Input', {
							hint		: 'Search for "Batman"',
							height 		: 20,
							autogrow 	: true,
							maxHeight 	: 100,
							plugins		: {
								Autocomplete : opts // Bind previous options //
							}
						});

						var input = $( 'input.__example3').Pure('Input'),
							srcUpdated;


						// Button click changes the source and clears previous history //

						$('button.__example3').click( function(){

							if( srcUpdated ){
								return;
							}

							$( 'input.__example3').Pure('Input:Autocomplete', {
								src : fbSrc
							});

							var auto = $( 'input.__example3').Pure('Input:Autocomplete');

							auto.clearHistory();

							input.setHint('Search for "Batman" on Facebook');

							srcUpdated = true;

						});

Input : Tokenizer

Options

Option Type Default
limit Integer Infinity
The maximum number of tokens that can be selected within the input.
on Object
An object listing associated listeners and their behaviour.

Listeners

Listener Arguments
addToken [ data, token ]
Listens for the addition of a token to the input.

data - an object containing the token's data

token - the public methods interface for the token

Example:


									$('.myInput').Pure('Input:Tokenizer', {
										on  : {
											addToken : function( data, token ){
												if( data.required ){
												
													// Prevent the token from being removed //
													token.attr( 'readonly', true );
												}
											}
										}
									});
removeToken [ data, token ]
Listens for the removal of a token from the input.

data - an object containing the token's data

token - the public methods interface for the token

Example:


									$('.myInput').Pure('Input:Tokenizer', {
										on  : {
											removeToken : function( data, token ){
												if( data.name == 'Captain America' ){
													alert( 'There is no Avengers without Captain America!' );
													
													// Reference the Tokenizer interface //
													this.clearTokens();
												}
											}
										}
									});
clickToken [ data, token ]
Listens for a click event on a token within the input.

data - an object containing the token's data

token - the public methods interface for the token

Example:

									
									$('.myInput').Pure('Input:Tokenizer', {
										on  : {
											clickToken : function( data, token ){
												var modified = $.extend( data, { /* Added elements */ });
												token.data( modified );
											}
										}
									});

Methods

Method Arguments Returns
getTokens Array of tokens

Get the tokens of the input as an Array.
setTokens Array

Set the tokens of the input.
clearTokens

Remove all the tokens from the input.
getTokenData Array of all the tokens' data

Get each token's data object pushed into a single array.
: remove

Remove the token. This method applies to a single token, for example a single item extracted from the array returned by getTokens
: attr [ attribute, boolean ]

Set the token to "readonly". This allows the user to navigate around it, but does not allow removal. This method applies to a single token.
: data [ data ]

Get or set the data of the token. It may be useful to store additional data than just what is collected from Autocomplete, for example, if the user is able to modify the data. This method applies to a single token.

Examples

						$('input.__example5').Pure( 'Input', {
							hint	: 'Search for "man" or "the"...',
							plugins		: {
								Tokenizer	: {
									limit	: 5,
									on		: {
										addToken	: function( data, token ){
											
											for( var x in tokens ){

												// Perform some kind of object comparison
												// to prevent duplicates

												if( isEqual( tokens[x].data(), token.data() ) ){
													token.remove();
													break;
												}

												tokens = this.getTokens();

											}
										},
										removeToken		: function(){
											tokens = this.getTokens();
										}
									}
								}
							}
						});

						var tokenizer	= $('input.__example5').Pure('Input:Tokenizer');

						tokenizer.setTokens([{
							name	: 'Ironman',
							id		: 1
						}]);

						var tokens = tokenizer.getTokens();
						
						
						$('button.__example5').click( function(){
							tokenizer.options({limit : 1});
						});

Input : Validation

Options

Option Type Default
filters Array

An array of custom functions that handle validation of the input. Each function has the value of the input, and tokens (if applicable) as arguments, and must return an object as its format, see below.

									function( value, tokens ){
										return {
											success : false,
											error	: 'Failed to validate this field'
										};
									}
delay Integer 0
The delay after the keypress event before validation is run.
events Array ['blur', 'keypress']
An array of events indicating when validation is to occur. Supported events are blur, keypress.
on Object
An object listing associated listeners and their behaviour.

Listeners

Listener Arguments
validate [ valid ]
Listens for the validation of the input based on validation methods and events defined in options.

valid - boolean describing state of input's validity

Example:

									
									$('.myInput').Pure('Input:Validate', {
										on  : {
											validate : function( valid ){
												if( ! valid ){
													$('.myInput').addClass('warning');
												} else {
													$('.myInput').removeClass('warning');
												}
											}
										}
									});
clear
Listens for the clearing of the input of its value and tokens.

Example:

									
									$('.myInput').Pure('Input:Validate', {
										on  : {
											clear : function(){
												$('.myInput').removeClass('warning');
											}
										}
									});

Methods

Method Arguments Returns
validate Object

Validate the input according to filters defined in options. Useful for applying during submission of a form. Returns an object specifying validity and the error if it's invalid. 

									var validator = $('.myInput').Pure('Input:Validator');
									var obj = validator.validate();
									
									/*	obj ==
									*
									*	{
									*		valid	: false,
									*		error	: 'Not a valid email address.'
									*	}
									*/

Examples


						$('input.__example7').Pure( 'Input', {
							plugins		: {
								Tokenizer : {
									on		: {
										addToken	: function(){
											$('input.__example7').Pure('Input:Validator').validate();
										}
									},
									limit	: 1
								},
								Validator : {
									filters : [
										function( value, tokens ){
											var valid = false;
											for( var x in tokens ){
												if( tokens[x].data().name == 'Ironman' ){
													valid = true;
													break;
												}
											}

											return {
												valid	: valid,
												error	: 'You did not choose Ironman.'
											}
										}
									],

									// Change element styling on validation //
									on		: {
										validate : function( valid, msg ){
											var $wrapper = $('div.pureui-8.pureui-input-mainWrapper')
															.first();
											if( ! valid ){
												$wrapper.addClass('error');
											} else {
												$wrapper.removeClass('error');
											}
										},
										clear	: function(){
											var $wrapper = $('div.pureui-8.pureui-input-mainWrapper')
															.first();
											$wrapper.removeClass('error');
										}
									},

									// Reset events - validation can only occur manually. //
									events : []
								}
							}
						});
